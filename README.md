# Personalized Movie Curator: Safe & Stable Recommendation System

본 프로젝트는 고객의 안정적인 만족과 플랫폼 체류 시간 증가를 최우선으로 고려한 영화 추천 시스템입니다. 넷플릭스의 CEE(Continuous Explore Exploit) 프레임워크를 바탕으로 "사용자가 싫어할 만한 것은 절대 추천하지 않는다(Do No Harm)"는 철학을 시스템 전반에 구현했습니다.

## 1. Dataset & Preprocessing

* **데이터셋:** [MovieLens 1M Dataset](https://grouplens.org/datasets/movielens/1m/)
* **전처리 및 레이블링:** 사용자, 영화 메타데이터, 평점 데이터를 병합했습니다. 평점 4점 이상을 긍정(1.0), 3점 이하를 부정(0.0)으로 변환하여 Implicit Feedback 기반의 이진 분류 문제로 재정의했습니다. 모델 입력을 위해 범주형 변수는 Label Encoding을 수행했습니다.

## 2. Model Architecture: AutoInt+

희소한 범주형 데이터 간의 복잡한 상호작용을 포착하기 위해 멀티 헤드 셀프 어텐션(AutoInt)과 딥러닝(MLP)을 결합한 **AutoInt+** 모델을 채택했습니다.

**임베딩 차원 (16):** 1M 데이터셋 규모에서 정보 손실과 과적합을 동시에 방지하는 최적의 차원 크기입니다.
**어텐션 층 (3 Layers)** : 어텐션 층은 유저의 나이, 직업, 영화 장르 등 서로 다른 특징들이 결합되어 발생하는 고차원적인 상호작용을 모델이 직접적으로 학습하게 합니다. AutoInt 논문 실험 결과에 따르면 3층 구조가 4차($4^{th}$-order) 결합까지 효과적으로 포착하며, 그 이상의 층은 데이터 대비 모델의 복잡도를 높여 과적합을 유발할 수 있어 3층으로 최적화했습니다.
**헤드 개수 (2 Heads)**: 멀티 헤드 구조는 영화의 '장르적 관계'와 유저의 '인구통계적 특성' 등 서로 다른 관점의 특징 조합을 독립적인 하위 공간에서 동시에 분석합니다. 데이터셋의 규모를 고려할 때 2개의 헤드를 사용하여 연산 효율성을 유지하면서도, 특징들 간의 복합적인 관계를 충분히 풍부하게 추출하도록 설계했습니다.
**MLP 은닉층 (32, 32):** 어텐션 층에서 이미 핵심 상호작용을 추출했으므로, 파라미터 낭비와 과적합을 막기 위해 얕고 가벼운 2층 구조로 설계했습니다.
**추론 속도 최적화:** Streamlit 웹 환경의 지연을 막기 위해 `model.predict()` 대신 `tensor_features = tf.constant(..., dtype=tf.int64)` 변환 후 `model()`을 직접 호출하여 추론 속도를 최적화했습니다.

## 3. Training Results & Metrics Evaluation

* **Mean NDCG@10:** 0.00511
* **Mean HitRate@10:** 0.01639

**메트릭 해석 및 활용:** 수치가 절대적으로 낮아 보일 수 있으나, 이는 평점 4점 이상만을 정답으로 간주하는 희소성을 가진 Implicit Feedback 환경에서 Top-K 추천을 수행했기 때문입니다. 수천 개의 영화 중 사용자가 향후 시청할 정확한 10개를 맞추는 엄격한 기준이 적용된 결과입니다.
본 시스템에서 모델의 예측 점수는 절대적인 추천 기준이 아닙니다. 이 점수는 철저한 하드 필터링과 '7:2:1 추천 로직'을 통과한 안전한 영화 풀 내에서 최종 랭킹을 산정하는 척도로만 활용됩니다.

# 4-1. Reommendation
초기 모델 예측 점수가 높은 순으로만 리스트를 구성했을 때, 사용자별 선호도가 다름에도 불구하고 특정 영화들만 중복 추천되는 현상이 발견되었습니다.

이는 학습 데이터의 불균형으로 인해 모델이 예측 성공률을 높이기 위해 확률이 높은 안전한 선택지만을 지향하는 경향이 있기 때문이었습니다. 특히 대중적 선호도가 높은 특정 영화들에 추천이 쏠리는 Popularity Bias 문제가 두드러지게 나타났습니다.

## 4-2. Personalized Recommendation Strategy: 7-2-1 Mix
용자가 자신의 개별적인 취향을 충분히 이해받고 있다고 느낄 수 있는 개인화된 추천을 제공하기 위해, 다음과 같은 4-2 (7:2:1 Mix) 전략을 통해 추천 로직을 정교하게 고도화하였습니다.

### 유연한 장르 콤보 시스템
전처리된 데이터 내 영화들은 최대 3개의 장르 정보를 가지고 있습니다. 단일 장르(Genre 1)만으로는 포착하기 어려운 사용자의 세밀한 복합 취향을 반영하기 위해, 영화별로 최대 2개의 장르를 조합한 **장르 조합**를 생성하여 분석 단위로 사용합니다.

- 장르의 순서와 상관없이 동일한 취향으로 간주합니다. (예: Romance + Comedy와 Comedy + Romance는 동일한 조합으로 취급)

- 구성 예시: Drama(단일), Drama + Comedy(복합), Action + Adventure(복합) 등

**Top 5 선정 방식:** 사용자가 과거 평점 4점 이상을 부여한 영화들의 장르 콤보 등장 빈도수를 계산하여 상위 5개 조합을 추출합니다. 이 Top 5를 기준으로 아래의 7:2:1 믹스를 구성합니다.

| 그룹 | 개수 | 선정 기준 | 기대 효과 |
| --- | --- | --- | --- |
| **A. 확실한 취향** | 7개 | 유저의 **Top 5 장르 콤보** 내에서 최고점 순 선발<br>

<br>*(대중픽 쏠림 방지를 위해 동일 콤보는 최대 3개로 제한)* | 확실한 만족감 (Exploit) |
| **B. 안전한 브릿지** | 2개 | 유저 선호 **Top 5 장르 + 대중적인 Tier 1/2 장르** 조합 | 성공률 높은 확장의 제공 (Safe Bridge) |
| **C. 새로운 장르** | 1개 | 안 본 조합 중 AI 예측 최고점 선발<br>

<br>*(단, Top 5 장르나 Tier 1/2가 1개 이상 포함되어야 함)* | 심리적 이질감을 낮춘 탐색 (Explore) |

*장르 티어(Tier) 구분: Tier 1(대중적), Tier 2(활동적), Tier 3(무거운), Tier 4(확실한 취향), Tier 5(민감/위험)*

## 5. Hard Filtering & Fallback

모델이 예측을 수행하기 전, 사용자 경험을 저해하는 요소를 차단하는 필터링을 적용합니다.

* **명시적 불호:** 탐색 가능성을 열어두기 위해, 한두 개 영화의 낮은 평점으로 장르를 섣불리 배제하지 않습니다. 특정 장르가 포함된 영화를 **3회 이상 시청**했고, 그중 **80% 이상에 낮은 평점(3점 이하)**을 주었을 때만 누적된 데이터를 근거로 해당 장르를 영구 블랙리스트에 등록합니다. 이는 사용자 데이터가 누적에 따라 업데이트됩니다.
* **민감 장르 원천 차단:** Tier 5영화 (Horror, Children's) 장르는 사용자의 과거 시청 이력이 없을 경우 후보군에서 완전히 배제합니다.
* 위 기준에 의해 블랙리스트로 지정된 장르가 대상 영화의 장르1, 2, 3에 단 하나라도 포함되어 있다면 추천 풀에서 탈락합니다.
* **Fallback 설계:** 위와 같은 촘촘한 필터링으로 인해 7:2:1 조건을 만족하는 영화가 부족해질 수 있습니다. 이 경우, 블랙리스트를 통과한 후보군 내에서 모델 예측 점수 기반으로 영화를 선정하여, 예외 상황에서도 시스템 에러 없이 10개의 영화를 보여줍니다.


## 6. 프로젝트 회고와 향후 과제
이번 프로젝트를 진행하면서, 사용자 입장에서 어떤 추천 시스템이 좋은 것인지 생각해보는 계기가 되었습니다. 제가 생각하는 가장 좋은 추천시스템은 '이 서비스가 나를 이해하고 있다'는 생각이 드는 서비스입니다. 이 철학을 이 서비스에 구현해보려고 했습니다. 만족도에는 다양한 요소가 영향을 미치겠지만, 그중 가장 중요하게 고려한 것은 '사용자가 싫어하는 걸 하지 않는 것'입니다. 이걸 1순위로 두되, 탐색에 따른 즐거움과 잠재적 체류 시가 증가를 놓치지 않으려 했습니다.    

AutoInt+는 피처 간의 복잡한 상호작용을 포착하는 데 탁월하지만, 모델 점수만으로 예측했을 때에는 데이터 불균형으로 인한 Popularity Bias가 두드러지게 나타났습니다. 바뀐 7:2:1 방식은 모델의 결과값을 그대로 수용하는 대신, 도메인 지식과 유저 분석을 바탕으로 데이터를 재구성한 결과입니다. 모든 도메인에서 마찬가지이겠지만, 특히 추천시스템에서는 고객에 대한 분석과 공감이 중요하다는 것을 배웠습니다.    

이번 프로젝트에서는 추천 로직의 편향성을 사후에 보정하는 방식을 이용했습니다. 앞으로 더 발전시켜보고 싶은 방향은, 모델 학습 전 단게에서부터 데이터 불균형을 해결하는 것입니다. 샘플링 기법 고도화나 손실 함수 조정을 통해, Long-tail 데이터에서 편향 없이 작동하는 모델을 구현해볼 예정입니다.
